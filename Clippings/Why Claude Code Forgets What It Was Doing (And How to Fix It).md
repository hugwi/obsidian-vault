# Why Claude Code Forgets What It Was Doing (And How to Fix It)

![rw-book-cover](https://i.ytimg.com/vi/Nnf-iOXwsxc/sddefault.jpg?sqp=-oaymwEmCIAFEOAD8quKqQMa8AEB-AH-CIAC0AWKAgwIABABGD8gWShyMA8=&rs=AOn4CLA8RWv_ckR-dGrwr06L6Ue_NsNpdw)

## Metadata
- Author: [[Edward Blake]]
- Full Title: Why Claude Code Forgets What It Was Doing (And How to Fix It)
- Category: #articles
- Summary: Cloud code sessions have a limited memory called a context window, which fills up and triggers compaction that summarizes and discards detailed history. To avoid losing important task details, use task state files and session handoff files to keep key information outside the conversation. Deliberately splitting sessions with handoff files helps manage context better than relying on automatic compaction, ensuring smooth continuity and efficient use of memory.
